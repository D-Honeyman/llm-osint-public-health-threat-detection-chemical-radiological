# llm-osint-public-health-threat-detection-chemical-radiological
Dataset and evaluation code for benchmarking a large language model for detecting public health threats (chemical &amp; radiological) from open-source intelligence (OSINT) news articles

# LLM Evaluation for Detection of Chemical and Radiological Public Health Events

This repository contains the evaluation dataset and evaluation code used to assess the performance of a large language model (LLM) for detecting chemical and radiological public health events from news articles.

The repository accompanies the manuscript:

**Chemical and Radiation Threat Detection Using a Large Language Model**

The aim of this repository is to enable reproducibility of the evaluation framework and provide a benchmark dataset for future research in event-based public health surveillance using large language models.

---

# Overview

Large language models are increasingly used for automated detection of emerging public health threats from open-source intelligence (OSINT) such as news media. However, reproducible evaluation frameworks for assessing their ability to extract structured information from such sources remain limited.

This repository provides:

- A **gold-standard annotated dataset** of 200 news articles
- **Human annotation outputs** (one file)
- **LLM extraction outputs** (same file as above)
- The **evaluation pipeline used to compute precision, recall, and F1 scores**

The repository allows independent researchers to reproduce the evaluation results reported in the manuscript.

---

# Dataset

The evaluation dataset contains **200 news articles** manually annotated by human reviewers.

Each row represents a single article.

Columns include:

| Column | Description |
|------|-------------|
| `_id` | Unique article identifier |
| `url` | Source article URL |
| `gpt4` | Structured extraction generated by the LLM |
| `feedback` | Human gold-standard annotation |

Annotations follow a structured bullet-list format representing entities extracted from each article.

---

# Entity Types Evaluated

The evaluation measures extraction performance for the following entities:

- Event identification
- CHEMICAL SUBSTANCE
- RADIOLOGICAL SUBSTANCE
- FATALITY COUNT
- CASE NUMBER
- COUNTRY
- STATE
- COUNTY
- CITY
- TIMEFRAME
- Years
- Months
- Dates
- date
- Dates or days of the week
- Adverbs

Two reporting transformations are applied in the evaluation script:

- `LOCATION` → **COUNTRY**
- `Answer` → **Event identification**

These transformations do **not alter the underlying annotations** but standardise reporting terminology.

---

# Evaluation Method

The evaluation script computes entity-level performance metrics:

- Precision
- Recall
- F1 score
- 95% Clopper–Pearson confidence intervals

The script performs:

1. Parsing of structured entity annotations
2. Label normalisation
3. Entity matching using:
   - exact matching
   - semantic similarity using sentence embeddings
4. Calculation of evaluation metrics

Embedding-based matching uses:
sentence-transformers/all-MiniLM-L6-v2

# Installation

Clone the repository:
git clone https://github.com/YOUR\_USERNAME/llm-event-detection-evaluation.git
cd llm-event-detection-evaluation

Install dependencies:
pip install -r requirements.txt

# Running the Evaluation

Run the evaluation script:
python scripts/ner_evaluation.py
--input data/evaluation_dataset_200_articles.xlsx
--output results/evaluation_results.csv
The script will generate:
evaluation_results.csv
containing the entity-level evaluation metrics.

# Reproducibility

The proprietary components used to collect and process the original news data (including Bing News API ingestion pipelines and large-scale scraping infrastructure) are not included in this repository due to institutional and contractual restrictions.

However, the following reproducible components are provided:

- the **LLM inference outputs**
- the **human gold-standard annotations**
- the **evaluation code**

These resources allow independent researchers to reproduce the evaluation results.
